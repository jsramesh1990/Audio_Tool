The **Yocto BSP build** for a Verdin iMX8M Plus is essentially a large automation pipeline. You don't build the kernel, U-Boot, and rootfs separately by hand. Instead, **BitBake** builds everything according to recipes and dependencies.

For a board from [Toradex](https://www.toradex.com?utm_source=chatgpt.com), the flow looks like this:

```text
                Yocto BSP

      +----------------------+
      |  meta-toradex        |
      |  meta-freescale      |
      |  meta-openembedded   |
      |  poky                |
      +----------------------+

                 |

             BitBake

                 |

    +------------+-------------+
    |            |             |
  U-Boot      Linux        RootFS
    |            |             |
  SPL         DTB          BusyBox
  ATF         Modules      Systemd
                           Qt
                           Docker

    +------------+-------------+

                 |

           Final Image

       .wic / TEZI Image

                 |

         Flash to eMMC
```

# 3. bblayers.conf

This file tells BitBake:

**Which layers are available?**

Example:

```bash
BBLAYERS ?=

poky/meta

poky/meta-poky

meta-openembedded/meta-oe

meta-openembedded/meta-python

meta-freescale

meta-toradex-bsp-common

meta-toradex-nxp
```

---

# 4. local.conf

This file tells BitBake:

**Which board and distro to build?**

Example:

```bash
MACHINE = "verdin-imx8mp"

DISTRO = "tdx-xwayland"
```

You can also select:

```bash
DISTRO = "poky"

DISTRO = "tdx-wayland"

DISTRO = "tdx-xwayland"
```

This is the standard workflow used in commercial projects based on Verdin iMX8M Plus. Once you understand this flow, the next step is usually learning **how to create your own Yocto layer (`meta-mycompany`)**, add custom recipes, and integrate your application into the BSP image.
